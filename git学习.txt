本地编辑了一些文件，下一步想要提交，首先需要pull，然后再提交，避免覆盖。
这里pull并不会覆盖本地修改的文件，会将远程仓库的更新合并到本地。

git pull --merge 和 git pull --rebase的区别：
merge会将远程分支的更新合并到当前分支，如果有冲突需要手动解决
rebase会将本地的提交放到远程分支的更新之后，形成一条线性的提交历史
例如：
    远程分支历史：A---B---C   (origin/main)
    本地分支历史：A---B---D   (main)
    如果是merge：
        A---B---C
             \   \
              D---M   (main)
    如果是rebase：
        A---B---C---D'   (main)

如果本地修改的文件和远程仓库的更新有冲突，rebase会暂停并提示解决冲突，解决后需要使用git rebase --continue。
如果希望保持提交历史的线性，可以选择rebase；如果希望保留分支的合并历史，可以选择merge。

一般的项目开发中，一般自己先开一个自己的分支，等自己开发完成后，再合并到主分支
具体步骤：
    idea左下角的git插件中，右击main有一个新建分支，比如新建分支"myFirstBranch"
    然后右击该分支，选择 签出 把工作区切换到该分支
    （这样下一次push才会push到这一分支，假如工作区还在main，那改的还是main！！！）
    不断地开发，push，这一分支没bug了就可以合并到main了，切换工作区到main以后，右键自己的分支，选择合并
    合并以后右键main，选择push推送。


当开发自己分支时，有他人的修改，此时的更新操作：
    左上角的版本控制点击更新项目，
    git插件中右键主分支，点击更新，现在主分支更新到最新了
    切换工作区到自己的分支，
    右键主分支，选择合并到当前分支，即可完成更新。
